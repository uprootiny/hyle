Coggy — Cognitive Architecture

Rust-native AtomSpace + PLN + ECAN. Zero dependencies.

v0.1 — Core (done)

  • AtomSpace hypergraph with indexed lookups
  • Atom types: ConceptNode, PredicateNode, InheritanceLink, EvaluationLink, ListLink
  • TruthValue (strength + confidence) with validity checks
  • AttentionValue (STI/LTI) per atom
  • ECAN attention: boost, spread, decay, rent
  • PLN forward chaining (deduction rule, depth-N)
  • NL→Atomese parser (is-a, has, evaluation patterns)
  • Base ontology (37 atoms: animal taxonomy, basic concepts)
  • Tikkun self-repair (5 diagnostics)
  • REPL with full cognitive traces

v0.2 — Inference & Memory (next)

  • Unit tests for all modules
  • --json flag for machine-readable traces
  • Abduction rule (backward inference)
  • Induction rule (pattern→generalization)
  • Persistent AtomSpace (serde → JSON/bincode)
  • LTI (long-term importance) with forgetting
  • Atom versioning (track TV changes over time)
  • Pattern matcher (find subgraph patterns)

v0.3 — LLM Integration

  • OpenRouter client for free model grounding
  • LLM→Atomese: parse LLM responses into atoms
  • Atomese→LLM: shape prompts from attention focus
  • Hybrid inference: PLN + LLM for gap-filling
  • Episodic memory (session traces as atoms)

v0.4 — Autonomy

  • Goal system (GoalNode with priority scheduling)
  • Action selection (pick next action from attention)
  • Self-modification (rewrite own ontology)
  • Multi-agent: Coggy↔Coggy communication via shared AtomSpace
  • Web dashboard API (JSON traces over HTTP)

Hyle — Code Assistant

Rust-native, OpenRouter-powered. Single binary.

v0.3 — Current (stable)

  • OpenRouter SSE streaming client
  • TUI with ratatui (model picker, traces, sparklines)
  • Session management with resume
  • Tool system: read, write, edit, bash, glob, grep
  • Multi-model fallback (8 free models)
  • Telemetry: CPU/memory pressure detection
  • hyle-api HTTP server for sketch submission
  • 289+ tests passing
  • Typed API errors with rate limit detection
  • Shared HTTP client (connection pooling)
