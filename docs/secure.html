[secure] hyle

Guardrails, audit trails, and defense in depth

Destructive Command Blocking

Pattern matching blocks rm -rf, fork bombs, disk overwrites before execution

Atomic File Writes

Write to temp file, fsync, rename. No partial writes. No corruption.

Write Verification

Read-back check after every write confirms content matches intent

Backup Rotation

Timestamped backups for every modified file. Last 3 versions kept.

Secure Config Storage

API key stored with 0600 permissions in XDG config directory

Tool Timeout

60s default timeout on all tool executions. Configurable.

Blocked Patterns

BLOCKED_PATTERNS in src/tools.rs

rm -rf Recursive force delete
rm -r / Root directory delete
:(){ :|:& }; Fork bomb
dd if=/dev/zero Disk overwrite
curl | sh Remote code execution
wget | bash Remote code execution

Write Flow

write(path, content)
    │
    ├── 1. Check file size limit (10MB)
    │
    ├── 2. Create backup: path.{timestamp}.bak
    │
    ├── 3. Write to temp file: path.tmp.{random}
    │
    ├── 4. fsync() temp file
    │
    ├── 5. rename() temp → path (atomic)
    │
    └── 6. Read back and verify content matches
